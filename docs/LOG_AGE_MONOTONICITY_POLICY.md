# LOG_AGE Monotonicity Policy

Policy version: `log_age_monotonicity.v1`

Default EFC jitter threshold: `0.00025` EFC

## Scope

This policy defines which Gate 2 interval rows are eligible for the first
capacity baseline ladder. It applies to reduced LOG_AGE exposure features and
the interval subset registry, not to raw LOG traceability claims.

No model training is authorized by this document alone. The policy authorizes
construction of clean/tolerant interval subsets that downstream baseline code
must use.

## Evidence Basis

The detailed LOG_AGE monotonicity report records default-tolerance violations in
`reports/audit/log_age_monotonicity_violations.parquet`; the committed summary
is `reports/audit/log_age_monotonicity_summary.csv`.

Current Gate 2b evidence:

- Strict LOG_AGE QA reports `7,107` timestamp/EFC decreases.
- The detailed default-tolerance report records `7,071` violations.
- The summary shows default-tolerance detailed violations are EFC decreases,
  not timestamp decreases.
- The largest top-20 detailed EFC drops are about `0.0002` EFC.
- `1,054` interval rows carry `LOG_AGE_monotonicity_violation`.

This pattern is treated as reduced-log quantization/jitter unless a timestamp
drop or large EFC drop is observed.

## Policy Rules

| Condition | Decision |
|---|---|
| Timestamp decrease | Hard fail for clean baselines. Exclude affected intervals unless traced and resolved. |
| EFC decrease `<= 0.00025` EFC and no timestamp decrease | Treat as small reduced-LOG jitter. Keep in tolerant baseline subset and mark `small_efc_jitter`. |
| EFC decrease `> 0.00025` EFC | Exclude from tolerant clean baselines pending raw LOG confirmation. |
| Missing LOG_AGE rows | Exclude from all clean baseline subsets. |
| Non-positive interval duration | Exclude from all clean baseline subsets. |
| Inserted diagnostics | Count and mask; never use `cap_aged_est_Ah`, `R0_mOhm`, or `R1_mOhm` as interval features unless explicitly prior-to-cutoff in a later policy. |

## Subsets

`baseline_clean_strict`:

- `duration_s > 0`
- `LOG_AGE_available == true`
- `log_age_row_count > 0`
- `log_age_monotonicity_violation_count == 0`

`baseline_clean_tolerant`:

- `duration_s > 0`
- `LOG_AGE_available == true`
- `log_age_row_count > 0`
- no timestamp decrease
- no EFC drop above `0.00025` EFC

`sensitivity_flagged_monotonicity`:

- true for any interval with `log_age_monotonicity_violation_count > 0`
- downstream baseline reports must include a sensitivity run excluding these
  intervals.

## Reporting Requirements

Every baseline report must include:

- number of intervals in the full interval table
- number of strict-clean intervals
- number of tolerant-clean intervals
- number of intervals excluded by timestamp decrease
- number of intervals excluded by large EFC drop
- number of intervals excluded by missing LOG_AGE
- number of intervals excluded by duration error
- number of intervals marked `small_efc_jitter`

The interval subset registry generated by `mbp split interval-subsets` is the
source of truth for these counts.
